# Module 1e Local Video Upload Design

## Goal
实现模块 1e 的本地视频上传 MVP：在“基本信息”模块中打开“商品视频”弹窗，固定走“本地上传”路径，选择本地 `.mp4` 文件，等待上传完成并点击“确定”。

## Scope
- 包含：视频文件路径解析、视频上传入口定位、本地上传 tab 选择、文件上传、上传完成确认、确认按钮提交
- 不包含：媒体中心选视频、视频封面替换、多个视频、视频删除/重传、系统 Finder 交互自动化

## Approaches
1. 直接命中页面 `input[type=file]` 并 `setInputFiles()`。推荐。
   - 优点：最稳定，绕过 macOS 原生文件选择器与 TCC 噪音；依然属于本地上传逻辑。
   - 风险：如果页面直到点击上传区后才懒加载 file input，需要补一个 filechooser fallback。
2. 强依赖原生文件选择器（点击上传区 -> Open）。
   - 优点：更贴近人工。
   - 风险：跨环境更脆，测试难度高，没必要作为 MVP 主链。
3. 继续沿用媒体中心。
   - 优点：复用旧文档。
   - 风险：已经偏离真实页面和当前用户要求，不采纳。

## Decision
采用方案 1，并保留方案 2 作为兜底：
- 新增 `fillVideo(page, data)`
- 支持 `video_file` 为绝对路径或项目相对路径
- 优先：弹窗内直接查找 `input[type=file]` 后 `setInputFiles()`
- 兜底：若 input 不存在，再点击上传区并监听 `filechooser`
- 上传成功的通过证据：预览/文件名可见 + `视频名称` 已填或自动生成 + `确定` 按钮可点击
- 任一步失败：截图并转人工，不炸掉后续模块

## Success Criteria
- `video_file` 为空时安全跳过
- 路径有效时能解析出绝对路径、文件名、标题名
- 测试 DOM 中能完成“本地上传 -> 确定”
- 真实页面能完成提供视频 `奔驰w221改装大灯总成.mp4` 的上传并确认
